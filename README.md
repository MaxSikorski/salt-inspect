# SALT Inspector — Industrial Defect Detection in Your Browser

**Real-time anomaly detection powered by SALT (Static-teacher Asymmetric Latent Training, Apple ICLR 2025), trained on MVTec AD, running entirely in your browser via ONNX Runtime Web.**

## Live Demo

> **[salt-inspector](https://maxsikorski.github.io/salt-inspect/)**

- Upload any image and see per-patch anomaly heatmaps
- Point your camera at objects for real-time defect detection
- Browse sample defects across 15 MVTec AD categories
- All inference runs client-side — no data leaves your device

## How It Works

SALT Inspector uses a two-stage self-supervised training pipeline:

```
Stage 1 — MAE Teacher (V-Pixel):
  ViT encoder learns "what normal looks like" via pixel reconstruction
  Trained on normal manufacturing images only

Stage 2 — JEPA Student (Frozen Teacher):
  Student encoder learns robust patch-level representations
  Frozen teacher provides learning signal — no EMA needed

Inference — Anomaly Detection:
  Encode image → 196 patch embeddings (14×14 grid)
  Compare each patch to normal reference embeddings
  Highlight anomalous regions with a heatmap overlay
```

## Architecture

| Component | Specification |
|-----------|---------------|
| Input | 224×224 RGB image |
| Backbone | ViT (192d, 12 layers, 3 heads) |
| Patches | 196 (14×14 grid, 16px each) |
| Output | Per-patch embeddings [1, 196, 192] |
| Model size | ~15-20MB ONNX (fp32) |
| Inference | <100ms in browser (WASM) |
| Dataset | MVTec AD (15 categories) |

## Key Innovation

SALT eliminates the exponential moving average (EMA) from JEPA training — the most complex and memory-intensive component. Instead, it freezes the teacher after Stage 1, enabling:

- **30% fewer FLOPs** than V-JEPA 2
- **"Weak teacher, strong student"** — small teacher trains large student
- **Student loss predicts downstream accuracy** (R² = 0.951)

Reference: [SALT paper (arXiv:2509.24317)](https://arxiv.org/abs/2509.24317) — Apple Research, ICLR 2025
